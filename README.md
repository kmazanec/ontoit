# OntoIt — Agentic Tax-Filing Assistant

A warm, conversational web chat that turns a single W-2 into a completed
**2025 IRS Form 1040** you can download — built as an agentic harness whose four
pillars (stateful chat loop, real tools, code-enforced guardrails, live
observability) are enforced by a [LangGraph](https://langchain-ai.github.io/langgraph/)
graph, not by the prompt.

See [`docs/`](./docs) for the PRD, architecture (ADRs), and roadmap.

## Run locally (one command)

```bash
cp .env.example .env          # then put your Anthropic API key in .env
docker compose up
```

Open <http://localhost:8000>. The only required setting is `ANTHROPIC_API_KEY`.

### Without Docker

```bash
pip install -e .
export ANTHROPIC_API_KEY=sk-ant-...
uvicorn app.main:app --reload
```

## What works today (iteration 01 — walking skeleton)

- A minimal web chat with a **live observation panel** beside it.
- One-click **"Use the sample W-2"** (a realistic fake W-2 ships in `assets/`).
- A warm greeting from the agent, streamed over **SSE**, with each step shown as
  a structured **observation event** in the UI.
- Session state lives entirely in a **signed cookie** — nothing is stored
  server-side, so no real PII is persisted and the app survives a host restart.

Later iterations add W-2 extraction, the deterministic tax engine, the
≤5-question conversation, the filled 1040 PDF, deployment, and two stretch
panels (mid-conversation correction, a "show your work" tax trace).

## Deploy to Render

The repo ships a `render.yaml` Blueprint so the service can be reproduced from
scratch via Render's Blueprint feature.

### One-time setup

1. Push this repo to GitHub.
2. In the Render dashboard choose **New → Blueprint** and connect the repo.
   Render reads `render.yaml` and creates a Docker web service called **ontoit**.
3. Set the one required secret in the Render dashboard under the service's
   **Environment** tab:

   | Key | Value |
   |-----|-------|
   | `ANTHROPIC_API_KEY` | your Anthropic API key (never committed) |

   `SESSION_SECRET` is auto-generated by Render (`generateValue: true`) — you
   don't need to set it.

4. Trigger a first deploy (or let the Blueprint trigger it automatically).

### Manual (without Blueprint)

Create a **Web Service** from the repo, choose **Docker** runtime, leave the
Dockerfile path as `./Dockerfile`, set `ANTHROPIC_API_KEY` in Environment, and
use `/health` as the health-check path.

### Free-tier cold starts

Render's free tier spins a service down after ~15 minutes of inactivity.
The first request after spin-down incurs a 30-60 second cold start — a judge
hitting the URL cold should wait up to a minute before retrying.

To keep the service warm during judging, set `KEEP_ALIVE_URL` to the service's
own `/health` URL (e.g. `https://ontoit.onrender.com/health`). The app then
self-pings every 10 minutes. The more reliable option is an external uptime
pinger such as [cron-job.org](https://cron-job.org) hitting
`https://<your-service>.onrender.com/health` every 10 minutes — this survives
a service restart, whereas the self-ping stops with the process.

### Smoke-testing the live deployment

```bash
./scripts/smoke_test.sh https://<your-service>.onrender.com
```

The script exercises every endpoint end-to-end and prints PASS/FAIL per step.

OntoIt · Architecture · variant B

LangGraph Agent — the four pillars, enforced in the graph

Claude drives; the graph enforces. Nodes run intake → extract → validate → collect → compute → fill → done with a guardrail node and a validate → collect re-ask loop. Hover a pillar to light up the nodes & edges that implement it; click any node for how it’s enforced outside the LLM’s decision path.

Pillar 1 · Chat loop
Stateful graph + cookie
Conversation is real graph state, carried in a signed cookie — not a transcript the model re-reads.
Pillar 2 · Tools
Real tool calls
extract / compute / fill are genuine tool invocations the agent makes — not prose it generates.
Pillar 3 · Guardrails
Edges + nodes, in code
questions_asked ≥ 5 edge, guardrail node, and validate node — all enforced outside the LLM’s decision path.
Pillar 4 · Observation
Live SSE events
Every transition emits one ObservationEvent, streamed live to the UI.
hover a pillar = highlight implementing nodes/edges · click a node = enforcement detail · the pink dashed wires are the ObservationEvent funnel
Takeaway
A real LLM-driven agent whose limits are concrete code and edges a judge can point at — not prompt requests. The question budget is a conditional edge; guardrails and validation are nodes outside the model’s decision path; every transition streams an ObservationEvent live.

OntoIt · LangGraph agent · four pillars

A real agent whose limits are concrete code a judge can point at — not prompt requests.

Seven nodes, a guardrail node, and a re-ask loop. Hover a pillar to light up the part of the graph that proves it.
1
Pillar I

Chat Loop

A stateful graph drives the dialogue, and the whole conversation state lives in a signed cookie — message history plus extracted W-2 — so any node can resume the thread with no server-side session. intake · collect (stateful graph + cookie)

2
Pillar II

Tools

The agent's real work is three deterministic tools: extract (pdfplumber→Vision), compute (pure Decimal tax math), and fill (PyMuPDF onto the IRS form). Claude calls them; it doesn't do them. extract · compute · fill

3
Pillar III

Guardrails

A conditional edge — questions_asked >= 5 — plus a guardrail node and a validation node enforce the limits outside the LLM's path. The cap is code on an edge, not a sentence in a prompt. if questions_asked >= 5: stop

4
Pillar IV

Observation

Every transition emits an ObservationEvent, streamed live over SSE to the UI and fanned out to logs and /health. The agent's whole run is watchable as it happens. emit(ObservationEvent) on every edge
